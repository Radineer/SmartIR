## SmartIR現状の弱点（改善対象）

### 1. AnalysisResultが浅い
- **現状**: summary + 3値センチメント + key_points のみ
- **競合水準**: セグメント別分析、業績要因分析、前期比較、ガイダンス分析

### 2. LLMプロンプトが汎用的
- **現状**: 全文書に同一プロンプト（GPT-4）
- **競合水準**: 文書タイプ別（決算短信/有報/説明資料）で最適化

### 3. PDF解析が不完全
- **現状**: テキスト抽出のみ。表・図を無視
- **競合水準**: 表形式データ（PL/BS/CF）の構造化抽出

### 4. 予測と分析が未統合
- **現状**: ML予測とLLM分析が別パイプライン
- **競合水準**: 決算内容→株価インパクト予測の統合分析

### 5. 時系列比較なし
- **現状**: 単発分析のみ
- **競合水準**: 前四半期比較、通期進捗率、YoY比較

## 受入条件

### 機能要件

#### A. 分析スキーマ拡張
- [ ] AnalysisResultに以下のカラム追加:
  - `revenue_yoy` (Float) — 売上高前年同期比 (%)
  - `operating_profit_yoy` (Float) — 営業利益前年同期比 (%)
  - `net_profit_yoy` (Float) — 純利益前年同期比 (%)
  - `guidance_revision` (String: up/down/unchanged/none) — 業績予想修正方向
  - `guidance_detail` (JSON) — 修正内容詳細
  - `segments` (JSON) — セグメント別分析
  - `risk_factors` (JSON) — リスク要因リスト
  - `growth_drivers` (JSON) — 成長ドライバーリスト
  - `financial_metrics` (JSON) — PL/BS/CFの主要数値
  - `comparison_note` (Text) — 前期比較コメント
  - `stock_impact_prediction` (String: positive/negative/neutral)
  - `stock_impact_confidence` (Float: 0-1)
  - `analysis_depth` (String: quick/standard/deep) — 分析の深さ
  - `llm_model` (String) — 使用したモデル名
  - `processing_time_sec` (Float) — 処理時間

#### B. 多段階分析パイプライン
- [ ] Phase 1 (速報 ≤3分): タイトル+基本数値+方向性
- [ ] Phase 2 (標準 ≤15分): 全文要約+センチメント+キーポイント
- [ ] Phase 3 (深層 ≤60分): セグメント分析+時系列比較+予測統合

#### C. 構造化データ抽出
- [ ] 決算短信の表形式データ（PL/BS/CF）を自動抽出
- [ ] 前年同期比を自動計算
- [ ] 業績予想修正（上方/下方/据え置き）を自動検出
- [ ] セグメント別売上・利益を自動抽出

#### D. 文書タイプ別分析プロンプト
- [ ] 決算短信用プロンプト（数値+予想修正フォーカス）
- [ ] 有価証券報告書用プロンプト（リスク+事業詳細フォーカス）
- [ ] 決算説明資料用プロンプト（戦略+見通しフォーカス）
- [ ] 適時開示用プロンプト（速報+インパクト評価）

#### E. 予測精度トラッキング
- [ ] 過去の株価予測と実結果の照合テーブル
- [ ] 的中率の自動計算・公開
- [ ] 精度に基づくモデル自動選択

#### F. 時系列比較
- [ ] 同一企業の過去分析結果との比較
- [ ] 四半期ごとの業績推移トラッキング
- [ ] 通期業績予想に対する進捗率計算

### 非機能要件
- パフォーマンス: Phase 1速報は決算発表から3分以内
- 精度: センチメント分析の精度80%以上（人間評価ベース）
- コスト: Claude APIコスト月額5,000円以内（Sonnetメイン）

## 制約事項
- フレームワーク: Python + SQLAlchemy + GitHub Actions
- AI: Claude API (Sonnet/Haiku) メイン、GPT-4はフォールバック
- データ永続化: Supabase PostgreSQL
- PDF解析: pdfminer + Claude Vision（表の構造化）
